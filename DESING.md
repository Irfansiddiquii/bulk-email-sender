# DESIGN PRINCIPLES

## Goal

Build a premium production-ready SaaS dashboard.

The interface should feel comparable in quality to products like:

- Linear
- Vercel
- Resend
- Clerk
- Notion
- GitHub Dashboard

Do NOT copy these products.
Use them only as quality references.

---

# Design Style

- Modern
- Minimal
- Premium
- Enterprise
- Clean
- Spacious

Avoid:

- Generic templates
- Bootstrap style UI
- Flat boring cards
- Oversized buttons
- Inconsistent spacing

---

# Color System

Primary:
Indigo / Blue

Accent:
Emerald

Danger:
Rose

Warning:
Amber

Success:
Green

Background:
Slate

Cards:
White with subtle shadow

Never use random colors.

---

# Typography

Use clear visual hierarchy.

Large headings.

Comfortable spacing.

Readable font sizes.

Avoid walls of text.

---

# Components

Every component should have:

- hover state
- active state
- disabled state
- loading state
- empty state
- error state

---

# Forms

Professional forms.

Floating labels where appropriate.

Clear validation.

Helpful errors.

Keyboard accessible.

---

# Tables

Professional tables.

Support:

- Search
- Pagination
- Sorting
- Empty state
- Loading skeleton

---

# Dashboard

Cards should include:

- icon
- title
- value
- trend
- subtle animation

---

# Sidebar

Modern dark sidebar.

Active item highlight.

Smooth hover.

Collapsible.

---

# Animation

Keep animations subtle.

Prefer:

- fade
- scale
- slide

Avoid flashy animations.

---

# Responsive

Support:

Desktop

Laptop

Tablet

Mobile

Never break layouts.

---

# Accessibility

Always:

- semantic HTML
- labels
- aria attributes
- keyboard navigation
- color contrast

---

# Performance

Prefer reusable components.

Avoid duplicated UI.

Keep layouts lightweight.
